## Implemented Features

### 1. File Watcher (`forge-core/src/file_watcher.rs`)

**Key Features:**
- Uses `notify` crate (v6.1) for file system monitoring
- 500ms debounce window to handle rapid edits
- Incremental updates via `upsert_file` and `remove_file`
- NO full reindex (performance critical requirement met)
- Handles Create, Modify, and Remove events
- Async event processing with tokio

**Critical Design Decisions:**
- Uses channel-based event delivery
- Debouncing prevents excessive updates during rapid edits
- Graceful error handling for file read failures
- Skips directory events (only processes files)

**Code Snippet:**
```rust
pub struct FileWatcher {
    context_index: Arc<Mutex<dyn ContextIndex>>,
    watch_path: PathBuf,
    debounce_timeout: Duration,
    last_event_time: Arc<Mutex<Option<std::time::Instant>>>,
}
```

### 2. Verify-Symbol-Before-Edit (`forge-core/src/event_loop.rs`)

**Key Features:**
- Integrated into `tool_diff_edit` method
- Calls `resolve_symbol` BEFORE applying edit
- REJECTS edits with non-existent symbols
- Clear error messages indicating rejection
- Prevents #3 hallucination (editing non-existent APIs)

**Symbol Extraction:**
- Basic pattern matching for function calls: `function_name(`
- Method call detection: `TypeName::method(`
- Skips symbols already present in old_text (rearrangements)

**Critical Design Decisions:**
- Only checks NEW symbols (not ones in old_text)
- Returns descriptive error messages
- Does not prevent edits to existing code
- Can be enabled/disabled via `with_context_index()`

**Code Snippet:**
```rust
async fn tool_diff_edit(&self, tool_call: ToolCall) -> Result<()> {
    // ... extract parameters ...

    if let Some(context_index) = &self.context_index {
        self.verify_symbols_before_edit(context_index, &old_text, &new_text).await?;
    }

    self.sandbox.diff_edit(&path, &old_text, &new_text).await?;
    Ok(())
}
```

## Performance Characteristics

### File Watcher:
- **Memory:** O(1) per watched directory
- **CPU:** Event-driven (no polling overhead)
- **Debounce:** 500ms window prevents excessive updates
- **Incremental:** NO full reindex (requirement met)

### Symbol Verification:
- **Overhead:** O(n) where n = number of symbols in edit
- **Blocking:** Minimal (only during diff_edit tool calls)
- **Cache:** ContextIndex caches symbols in memory

### Mock ContextIndex:
- **Memory:** O(total symbols + total files)
- **Insert:** O(1) per symbol
- **Query:** O(n) where n = total symbols (linear scan)
- **Resolve:** O(1) HashMap lookup

## Known Limitations

### Mock ContextIndex:
1. **Basic Symbol Extraction:** Only handles simple patterns
2. **No Semantic Search:** Returns all chunks (no ranking)
3. **No Tree-sitter:** Uses regex-like patterns
4. **Temporary:** Will be replaced by Track A's SemanticContextEngine

### Symbol Verification:
1. **Simple Extraction:** May miss complex patterns
2. **False Positives:** Could reject valid code in edge cases
3. **No Context:** Doesn't understand scope/imports

### File Watcher:
1. **Platform Dependent:** notify crate behavior varies by OS
2. **Network Drives:** May not work reliably on network filesystems
3. **Rapid Edits:** Debouncing may merge very rapid changes

## Integration with Track A

### ContextIndex Trait:
Both tracks share this frozen contract:
```rust
pub trait ContextIndex {
    fn upsert_file(&mut self, path: &Path, src: &str);
    fn remove_file(&mut self, path: &Path);
    fn query(&self, q: &str, k: usize) -> Vec<CodeChunk>;
    fn resolve_symbol(&self, name: &str) -> Option<Symbol>;
}
```

### Migration Path:
1. Track A implements `SemanticContextEngine` with `ContextIndex`
2. Track B replaces `MockContextIndex` with `SemanticContextEngine`
3. No code changes needed in Track B (trait-based polymorphism)
4. Single line change:
   ```rust
   // Before:
   let index = Arc::new(Mutex::new(MockContextIndex::new() as dyn ContextIndex));

   // After (Track A integration):
   let index = Arc::new(Mutex::new(SemanticContextEngine::new() as dyn ContextIndex));
   ```

## Example Usage

### Setting up File Watcher:
```rust
let mut context_index = MockContextIndex::new();
let index = Arc::new(Mutex::new(context_index as dyn ContextIndex));

let mut watcher = FileWatcher::new(

    500, // 500ms debounce

watcher.watch()?;
```

let provider = AnthropicProvider::new(api_key, model)?;
