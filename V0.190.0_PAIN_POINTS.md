# Forge v0.190.0 Features → Pain Points Mapping

## Pain Point #8: Lock-in into single model/cloud
**Problem**: Users locked into Anthropic/Claude, can't use OpenAI/Gemini/local models.

**v0.190.0 Solution**:
- `ModelProvider` trait with 4 implementations:
  - `AnthropicProvider` (existing from v0.100.0)
  - `OpenAIProvider` (new) - GPT-4 support
  - `GeminiProvider` (new) - Gemini Pro support
  - `LocalProvider` (new) - Ollama/llama.cpp support
- Environment variable configuration per provider:
  - `ANTHROPIC_API_KEY` for Anthropic
  - `OPENAI_API_KEY` for OpenAI
  - `GEMINI_API_KEY` for Gemini
  - No API key required for local models
- Provider switching in forge.toml without code changes
- Fallback support: `provider.fallback = ["openai", "local"]`

**Evidence**:
- User can switch providers in forge.toml: `provider.default = "openai"`
- Can run models locally: `provider.default = "local"` with Ollama
- Fallback ensures redundancy: primary provider fails → automatic fallback
- All providers implement same `ModelProvider` trait → consistent interface

**Delivered Files**:
- `forge-provider/src/openai.rs` (150 lines)
- `forge-provider/src/gemini.rs` (135 lines)
- `forge-provider/src/local.rs` (145 lines)
- 9 provider tests passing

## Pain Point #10: AGENTS.md inconsistent across models
**Problem**: AGENTS.md format benefits depend on model, causing confusion and inconsistent behavior.

**v0.190.0 Solution**:
- **Layered Discovery**: Find AGENTS.md from current dir up to root, nearest wins
  - `AgentsDiscovery::discover_from(&start_dir)` returns files ordered nearest → root
  - Child directory AGENTS.md takes precedence over parent
  - Mimics how .gitignore works (nearest wins)
- **Optional/Advisory Treatment**: Never hard-fail if AGENTS.md missing
  - `has_agents_file()` returns bool (no panics)
  - Empty result returned if no AGENTS.md found
  - Graceful degradation in `ContextEngine::load_agents_md()`
- **Scope Field**: Optional `scope: local|global` in frontmatter
  - Default scope: `global` if not specified
  - Enables layered configuration (local overrides global)

**Evidence**:
- User can have repo-level AGENTS.md + directory-specific overrides
- Missing AGENTS.md doesn't crash Forge (returns empty string)
- Nested projects work correctly: `/project/subdir/AGENTS.md` wins over `/project/AGENTS.md`
- Cross-model consistency: All models use same discovery logic

**Delivered Files**:
- `forge-context/src/agents.rs` (215 lines)
- 7 AGENTS.md tests passing
- Integration with existing `ContextEngine::load_agents_md()`

**Example Usage**:
```bash
# Project structure
/my-project/
  AGENTS.md          # Global agents instructions
  /backend/
    AGENTS.md          # Backend-specific (wins)

# Forge running in /my-project/backend/
# → Loads backend AGENTS.md (nearest wins)
```

## Additional v0.190.0 Benefits

### MCP Client/Server
- **Benefit**: Integration with MCP ecosystem (40+ servers for filesystem, search, databases, etc.)
- **Relevance**: Future-proofs Forge for agentic ecosystem (Agentic AI Foundation standard)
- **Evidence**: `McpClient::list_tools()`, `McpClient::call_tool()` implemented

### Hooks System
- **Benefit**: User customization at lifecycle points (notify on task complete, custom verify, etc.)
- **Relevance**: Production workflows, CI/CD integration, monitoring
- **Evidence**: HookRegistry loads from forge.toml, 4 lifecycle events defined

### Skills Framework
- **Benefit**: Reusable prompt + tool combinations (e.g., "security-audit", "refactor-code")
- **Relevance**: Shareable workflows, community skills library, consistency
- **Evidence**: `SkillDiscovery::discover()` loads .md files from skills/ directory

### Headless Exec Mode
- **Benefit**: CI/CD integration, automated task execution
- **Relevance**: `forge exec "fix bug"` in GitHub Actions, exits 1 if verify fails
- **Evidence**: `ExecResult::exit_code()` returns 1 on verify failure

### Observability
- **Benefit**: Debug long-running tasks, track token usage, production monitoring
- **Relevance**: Cost tracking, performance analysis, debugging
- **Evidence**: `MetricsCollector::record_tokens()`, `TraceLog::write_to_file()`
