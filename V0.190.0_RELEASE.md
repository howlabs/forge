# Forge v0.190.0 Release Notes

## Overview
v0.190.0 completes the framework milestone, making Forge open and production-ready with MCP client/server, hooks, skills, multi-provider support, headless exec mode, and observability.

## New Features

### 1. MCP Client & Server (✅ Pain #8 - Lock-in)
- **MCP Client**: Connect to external MCP servers and call their tools from inside the agent loop
  - JSON-RPC 2.0 over stdio transport
  - Tools/list and tools/call support
  - Server subprocess management
- **MCP Server**: Expose Forge's own tools over MCP protocol
  - Tool registration and discovery
  - Request handlers for initialize, tools/list, tools/call
  - Error handling for unknown tools/methods
- **Module**: `forge-ext::mcp`
- **Tests**: 21 tests passing

### 2. Hooks System (✅ Lifecycle Events)
- **Lifecycle Events**: TaskCreated, TaskCompleted, PreEdit, PostVerify
  - Event payload serialization with serde
  - Event type discrimination with tagged enums
- **Hook Registry**: Load user hook scripts from forge.toml
  - Hook script path configuration
  - Hook triggering with event payload as stdin
  - Graceful handling of missing hook scripts
- **Module**: `forge-ext::hooks`
- **Tests**: 14 tests passing

### 3. Skills Framework (✅ Reusable Prompts)
- **Skill Types**: Frontmatter + markdown format
  - YAML frontmatter with name, description, tools
  - Markdown prompt content
  - Tool dependencies specification
- **Skill Discovery**: Load from skills/ directory
  - Automatic discovery of .md skill files
  - Skill retrieval by name
  - Empty directory handling
- **Module**: `forge-ext::skills`
- **Tests**: 12 tests passing

### 4. Multi-Provider Support (✅ Pain #8 - Lock-in)
- **OpenAI Provider**: GPT-4 support
  - Message conversion for OpenAI API format
  - Chat completions endpoint
  - Error handling
- **Gemini Provider**: Gemini Pro support
  - Message format adaptation (no system role)
  - Generative Language API integration
  - Content extraction from candidates
- **Local Provider**: Ollama/llama.cpp support
  - Ollama chat API format
  - llama.cpp completion format
  - Backend abstraction (Ollama/LlamaCpp)
- **All providers**: Implement ModelProvider trait
- **Module**: `forge-provider::{OpenAI,Gemini,Local}Provider`
- **Tests**: 9 tests passing (11 total in forge-provider)

### 5. Headless Exec Mode (✅ CI/CD Ready)
- **forge exec command**: Non-interactive execution
  - `forge exec "<task>"` with required --api-key
  - Machine-readable JSON output
  - Exit code 0 on success, 1 on verify failure
  - Text format option for human-readable output
- **ExecResult**: Structured result with task_id, duration_ms, steps_completed, verify_passed
- **Module**: `forge-cli::exec`
- **Tests**: 7 tests passing

### 6. Observability (✅ Production Monitoring)
- **Metrics Collection**: Token usage tracking
  - Per-provider token usage (input/output)
  - Total usage aggregation
  - Provider listing
- **Structured Tracing**: Span builders for operations
  - step_span for agent steps
  - tool_span for tool calls
  - provider_span for provider requests
  - hook_span for hook execution
  - mcp_span for MCP operations
- **Trace Log Export**: JSON run logs
  - Timestamped event recording
  - Event type and data
  - File export functionality
- **Module**: `forge-ext::observability`
- **Tests**: 15 tests passing

### 7. AGENTS.md Layered Discovery (✅ Pain #10 - Cross-Model)
- **Layered Discovery**: Find AGENTS.md from current dir up to root
  - Nearest file wins (layered discovery)
  - Upward traversal with parent directory resolution
- **Optional/Advisory Treatment**: Never hard-fail if missing
  - Returns empty list if no AGENTS.md found
  - Optional scope field in frontmatter (local/global)
  - Graceful degradation when files missing
- **AgentsFile**: Parsed structure with content and scope
- **Module**: `forge-context::agents`
- **Tests**: 7 tests passing

## Breaking Changes
- forge.toml format updated (new [provider], [hooks], [skills], [mcp] sections)
- Provider selection via environment variables (ANTHROPIC_API_KEY, OPENAI_API_KEY, GEMINI_API_KEY)
- CLI now requires subcommand: `forge repl` or `forge exec`

## Migration Guide
1. Update forge.toml to v0.190.0 format
2. Use `forge repl` for interactive mode (default behavior preserved)
3. Use `forge exec "<task>"` for CI/CD automation
4. Set environment variables for provider API keys
